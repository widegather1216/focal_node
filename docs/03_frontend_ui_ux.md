# [Doc 3] 프론트엔드 아키텍처 및 UI/UX 설계

이 문서는 React 18, TypeScript, Tailwind CSS, Tauri API를 결합한 데스크탑 앱의 프론트엔드 설계 및 사용자 인터페이스(UI) 최적화 전략을 다룹니다.

---

## 1. 상태 관리 및 성능 최적화 전략

로컬 데스크탑 앱 환경에서 수만 장 이상의 대용량 사진 라이브러리를 끊김 없이(Jank-free) 탐색하기 위해 프론트엔드는 다음 두 가지 핵심 라이브러리로 상태를 분리해 관리합니다.

### 1.1. 서버 상태 관리 (TanStack Query)
* **목적:** 사이드카 백엔드 API로부터 불러오는 비동기 데이터를 캐싱하고 동기화합니다.
* **주요 역할:**
  * **무한 스크롤(Infinite Scroll):** 대량 사진 조회를 위해 페이징 데이터 캐싱 및 스크롤 위치 기반 추가 로딩 처리.
  * **검색 결과 관리:** 사용자가 텍스트 검색 시 실시간 유사도 검색 및 필터링 결과를 캐싱하여 불필요한 네트워크 요청 감소.
  * **상세 메타데이터 패치:** 특정 사진 클릭 시 EXIF 및 AI 분석 결과를 최신 상태로 유지.

### 1.2. 로컬 전역 상태 관리 (Zustand)
* **목적:** 컴포넌트 간 유기적으로 실시간 반영되어야 하는 UI 상태를 저장합니다.
* **주요 역할:**
  * **검색어 및 하이브리드 필터 조건:** (기종, ISO 범위, 촬영 연도 등) 상태 보유 및 동기화.
  * **인덱싱 상태 감시:** Tauri Event 리스너로부터 전달받은 백그라운드 인덱싱 진행률(%) 및 현재 처리 중인 파일 정보를 전역으로 보관 및 업데이트.

### 1.3. 렌더링 최적화: 가상 스크롤 (Virtual Scroll)
* **라이브러리:** `@tanstack/react-virtual`
* **이유:** 수천 장의 사진을 뷰포트 내에 전부 HTML DOM으로 로드하면 심각한 메모리 부족 및 프레임 드랍이 발생합니다.
* **해결책:** 뷰포트에 표시되는 영역의 이미지 요소만 DOM에 남겨두고 나머지는 절대 좌표 배치를 통해 메모리 점유를 최소화합니다.

## 2. UI/UX 워크플로우 디자인

### 2.1. 온보딩 (Empty State)
* **시나리오:** 처음 앱을 실행하여 데이터가 비어 있을 때 노출되는 화면입니다.
* **구성:**
  * 화면 중앙에 입체감 있는 유리 효과(Glassmorphism) 스타일의 `[ + 사진 폴더 추가 ]` 버튼 배치.
  * 해당 버튼 클릭 시 Tauri의 네이티브 파일 탐색기 창(`@tauri-apps/api/dialog.open` API)을 호출하여 로컬 폴더 경로를 안전하게 가져옵니다.

### 2.2. 사이드바 (Navigation & Status)
* **구성:**
  * 폴더 트리 구조 탐색 기능.
  * 하단에 **인덱싱 진행 상태 패널** 배치:
    * **이벤트 기반 단방향 알림 구조:** 매초 API를 호출하는 폴링 방식 대신, Tauri 네이티브 이벤트 리스너(`@tauri-apps/api/event.listen`)를 등록하여 백엔드가 Rust Host를 통해 스트리밍 형태로 전송하는 `indexing-progress` 이벤트를 비동기로 실시간 수신합니다. 수신한 이벤트 데이터를 기반으로 진행률 프로그레스 바와 현재 처리 중인 파일명을 화면에 즉각 업데이트합니다.

### 2.3. 메인 검색 및 갤러리 뷰 (Main Content)
* **구성:**
  * **상단 바:** 세련된 자연어 검색창 제공. 우측의 필터 아이콘 클릭 시 드롭다운 패널이 확장되며 카메라 기종, 렌즈 기종, ISO 범위, 셔터스피드 범위, 촬영 일자를 세부 조정할 수 있는 UI 제공.
  * **하단 그리드:** 가상화 그리드로 정렬된 사진 리스트. 각 이미지 카드는 백엔드의 `GET /api/photos/{id}/thumbnail` 스트리밍 URL을 `src` 속성에 바인딩하여 백그라운드에서 지연 로드(Lazy Loading)합니다.

### 2.4. 상세 정보 패널 (Deep Dive Panel)
* **시나리오:** 갤러리에서 사진 카드 더블 클릭 또는 단일 클릭 시 우측에서 슬라이드 인(Slide-in) 형태로 등장하는 패널입니다.
* **구성:**
  * **고화질 렌더링:** RAW 파일인 경우에도 백엔드가 실시간 sRGB JPEG로 디코딩한 고해상도 이미지를 로드합니다.
  * **EXIF 배지 영역:** ISO, 셔터스피드, F값, 카메라 모델명 등을 칩(Chip) 형태의 세련된 UI 배지로 시각화합니다.
  * **AI 편집기:** Gemma 4 E4B-it가 추천한 캡션 및 태그가 표시됩니다. 사용자가 직접 텍스트를 수정하거나 태그를 추가/삭제하면 `is_user_edited` 필드가 `true`로 변경되며 SQLite DB에 즉시 업데이트를 요청합니다.
